The Challenge

An All-in-one Tool

One of Controlco's clients wanted an interface to manage the building automation operations of two of their facilities. Unlike our other projects, this one would be limited in terms of interactions with the data.

Our clients identified a need for a way to store floor plans and other shareable documents amongst their facilities engineers.

Because this was a brand new request, I had to research the best possible ways to design this. I looked at file upload systems and also interviewed several individuals.

Translating the Requirements

We fleshed out the requirements and had a working session on what the potential dashboard would look like. Since we didn't have all the data integrated, we agreed to focus on just designing the most essential part of the service; the homepage. The key requirements include:

  1. Dashboard homepage that will be an overview of systems
  2. Functionality where users can download and upload files
  3. Set of secondary navigation menu on homepage for building information, operational hours, contact list, as well as document repository

The Design Process

Wireframing

I built upon the client's mockup to create wireframes demonstrating the architecture and use case of the repository functionality.

We held an internal review with the engineers and data modelers to see if the design was feasible. This was the first time that we've introduced this feature, as none of our other clients needed this functionality. The engineering team had already began the backend operations of programming the repository while I was still working on the interactions and visual design.

Style Guide

The next step was to craft a design system for this template. The client gave me details about what colors they wanted the dashboard to be. After showcasing some combinations of typography and color guides, we were able to compromise on a palette of grays, as well as the conventional status colors.

Testing

Instead of using InVision to build out a clickable prototype, we went ahead and used DGLux5 to build a fully responsive and interactive platform. We tested the feature internally, and then allowed the client to access via public link.

To ensure that the design was scalable, we implemented a search feature, a vertical scroll, and sorting carets. If clients are adding a lot of documents to the repository, these features will help them find specific files.

The upload CTA appears only after a user chooses a file

Users will be able to download any file from the document library onto their local desktop

The Takeaway

Building the Foundation

I was challenged with the opportunity to craft a new feature that we can start using for other projects. I built a flexible system to get the product started but can expand if needed. For this use case, we introduced the feature that would let clients deal with uploading just one file at a time, as we were informed that there'd be less then ten files for each building. The client feedback was positive as they said that it catered to their customer's needs. As a result, we were contracted to implement this design for the several other properties in their portfolio.
