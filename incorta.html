Incorta

Data File Directory

Product Design

Users of the Incorta Data Analytics Platform were adding large amounts of data files into a single location with limited file management capabilities. I introduced a feature enhancement to store files within a directory, solving a longstanding issue for users— poor data management.

opportunity

The Villain of this Story: A Poor File Management System

The current Incorta Data Files page was one of the few remnants of the outdated Angular framework which the platform was built on a few years ago. One of the outstanding company roadmap items was to port the Data Files page over to React, but enhancing the data functionalities was the more anticipated feature for users.

Users have long complained about the lack of support for data file management in the Incorta platform. Files could only be uploaded in a flat location, with no grouping or customization. There was also no bulk upload concept. Users wanted to spend less time managing/finding/uploading files so they can proceed with their other functions within Incorta.

research and discoveries

Understanding How Incorta Users Manage Data

The PM and Solutions Architects frequently visited customers and users onsite. I synced up with them to gather intel on our users. I was able to get a good picture of our users' challenges and needs. The end users are data analysts, and are responsible for bringing data into their team's Incorta instance. From there, schemas are built to create a visualization layer that provide key insights into their business.

User Story Map

There are two ways of bringing data into Incorta. First is by connecting to a database, and second, is through uploading CSV or Excel files. However, users were met with a few challenges when going with the file route.

The Frustrations of Data Files

"The 'New' button wasn't obvious to me at first."

Users would click here to add a data file or data source, but they overlooked the fact that they were on the wrong tab before adding.

"I can only upload one file at a time."

Users were frustrated that they couldn't save time with a bulk upload.

"There's no search feature to find files."

There could be hundreds of data files, but there was no built-in searchbar to locate them.

"I can't group my files."

Users were annoyed that there was no way to group files for better discoverability.

It was revealed that users have been asking for a better navigation structure for a while.

I realized if we introduced the concept of folders in this enhancement, we would be allowing users to nest folders inside one another. Users will need a way to traverse through the file path. The only part of our app that utilizes somewhat of a directory structure is the Content tab, where dashboards and folders can be grouped into folders. Users complained about our current implementation of the breadcrumb in the Content tab because it didn't provide a way for them to navigate back to folders in a path. Users wanted the transparency of knowing their location in the hierarchical structure so they can browse back to higher levels.

The current navigation pattern only showed the current file location.

Users had a few workarounds for some of these challenges.

To ensure that relevant files are clumped together in the listing page, users adhered to a naming convention before uploading. For example, files that were used together were named in alphanumeric order. It helped greatly for them, but it was extra work upfront.

There was no concept of pagination on this page and because everything was loading at once, it would take over five seconds for all data to appear. Users resorted to Ctrl+F to find what they're looking for, but this would be a challenge for users who can't recall the exact name of these files.

Data Analysts wanted to spend less time on menial tasks like uploading, finding, and sharing data files.

A Data Analyst's job requires them to interpret data so they can deliver key insights on how the business is doing. They spend their time identifying trends and patterns in data. What they don't want to do is spend a lot of time bringing in data and locating them. Users want to be able to customize their Incorta instance so that these files can be shared with other users who may need that same data.

Vision

Our goal was to rethink the Data Files to be more like a directory so that users can manage their files for better discoverability and shareability.

Criteria for Success

  1. The ability to create folders and rename folders
  2. Uploading folders
  3. Managing permissions to overwrite
  4. Allow "New" button to combine actions for data file and data source
  5. Folder permissions
  6. Move files/folders
  7. Improved file navigation

design

Redesigning the Breadcrumb Concept

I looked at a few industry leaders to get an idea on how each of them handled file storage and navigation. Google Drive, Onedrive, Dropbox, and Egnyte were a few that I drew ideas from. I created wireframes to communicate what the new file structure and functionalities would look like. The new breadcrumb component would show the root location, an option to look at multiple filders in that path, the previous folder, and the current level that the user is in. This would solve the issue of users not knowing their location in the hiearchical structure.

The proposed designs aligned with the requirements, but we were met with engineering constraints.

In a sync with the Server and Front-end teams, we learned that they couldn't accommodate some of the solutions. They had already allocated resources to some of the higher priority backend projects. Some of the things they couldn't deliver in time include:

  • Functionality to move folders because there was not enough time to do any engineering spikes to figure out technical deficits
  • Merging the upload folders and files actions into one prompt because the current endpoints don't support that
  • Folder-level permissions

"What if a user doesn't have permission to a certain folder, but has access to the file within it?"

We realized the complexity of folders and permissions. There wasn't enough investigation around the edge cases. As a result of our discussion, we agreed to move some of the enhancements to a V2. For the first release, we are only attaching permissions to files instead of folders. I updated the wireframes to limit the scope of actions and handed it off to Engineering.

The Final Design

the delivery and impact

Small Feature Enhancements, Huge Increase in Productivity

The Data File enhancements were delivered in the Incorta 4.6 release (check out the release notes). After the launch, PMs evaluated the enhancements when meeting with customers. Users shared positive feedback, saying that there was a sizable difference in the experience of their workflow. They were able to share top-level folders with peers and cut down time by a lot. They also appreciated that the Data Files page was finally redesigned to match the rest of the platform. Finally, they also expressed their excitement for the upcoming enhancements in the second phase.

Another achievement from this was that the breadcrumb design is going to be the standard implementation. The next phase for this project is currently in the works, which includes folder-specific permissions and moving folders around.
