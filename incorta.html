Incorta

Data File Directory

Product Design

The Incorta Data Analytics Platform is used by hundreds of companies, including some big Fortune 500 names to resolve their most complex data analytics challenges. Data analysts had a longstanding issue with managing their data. There were major limitations that made it hard to organize and find data files. For the upcoming release, I introduced a solution that treated the Incorta Data File tab as a directory, resulting in a significantly improved workflow for data analysts.

opportunity

The Villain of this Story: A Poor File Management System

Incorta is a data analytics software that aggregates large complex business data in real time, eliminating the need to reshape it via data warehousing. Users have long complained about Incorta's lack of support for data file management. Data files were limited to one upload at a time, there was no concept of file locations, and a built-in search function was nowhere to be seen. Additionally, the current Data Files page was one of the few remnants of the outdated Angular framework. It was a roadmap item for Engineering to port the page over to React.

As the lead designer for this project, I started off identifying areas of improvement and worked hand-in-hand with stakeholders like Product and Solutions Architects to provide enhancements to the data analysts' workstream.

research and discoveries

Understanding How Incorta Users Manage Data

The PM and Solutions Architects frequently visited customers and users onsite to learn about our users' experience with Incorta. The end users are data analysts, which fall under our Power User persona. They use advanced analytical tools that allow them to dive into the data and work with it in more depth so that they can create visualizations to be consumed by others. But in order to do that, data needs to be brought into their Incorta instance first.

User Story Map

A story map demonstrating the steps users needed to take to bring in their data.

There are two ways of adding data into Incorta. First is by connecting to a database, and second is through uploading CSV or Excel files. However, users were met with a few challenges when going with the file route.

The Frustrations of Data Files

"Once a file has been uploaded, you can't change the name."

It was a pain having to either remember the file name or to modify the file before upload.

"I can only upload one file at a time."

Users were frustrated that they couldn't save time with a bulk upload.

"There's no search feature to find files."

There could be hundreds of data files, but there was no built-in searchbar to locate them.

"I can't group my files."

Users were annoyed that there was no way to group files for better discoverability.

It was revealed that users have been asking for a better navigation structure for a while.

By introducing the concept of folders, users would need a way to traverse through the file path. The only part of our app that utilizes somewhat of a directory structure is the Content tab, where dashboards and folders can be grouped into folders. Users griped about our existing breadcrumb in the Content tab because it didn't provide a hierarchical structure to browse back to higher levels.

The current navigation pattern only showed the current file location.

Users had a few workarounds for some of these challenges.

To ensure that relevant files are clumped together in the listing page, users made sure to name their files a certain way before uploading. It helped them find files more easily, but it was an extra step upfront.

There was no concept of pagination on this page and because everything loaded at once, it would take over five seconds for all the data to appear. Users used Ctrl+F to find what they're looking for, but this would be a challenge for users who can't recall the exact names of these files.

Data Analysts wanted to spend less time on menial tasks so that they can start modelling their data.

A Data Analyst's main duties include interpreting data so they can deliver key insights on how the business is doing. What they don't want to do is spend a lot of wrangling with data due to lack of features like bulk uploading, searching, sorting, filtering, and folders. Users want to be able to customize their Incorta instance so that these files are discoverable for themselves, and for their peers.

Vision

Our goal was to rethink the Data Files to be more like a directory so that users can better manage their files for a more efficient process.

Criteria for Success

  1. Create/modify folders
  2. Upload folders
  3. Manage permissions to overwrite
  4. Allow "New" button to combine actions for data file and data source
  5. Folder permissions
  6. Move files/folders
  7. Improved file navigation

Redesigning the Breadcrumb Concept

I looked to a few leaders in the storage space like Google Drive, Onedrive, and Dropbox to get an idea on how to handle file management and navigation. In my design proposal, the new breadcrumb component would show a clear hierarchical structure, and easy access to files in that path.

The proposed designs aligned with the requirements, but were met with engineering constraints.

In a sync with the Server and Front-end teams, we learned that they couldn't commit to some of our asks. They had already allocated resources to some of the higher priority backend projects. Some of the things they couldn't accommodate in this sprint include:

  • Functionality to move folders
  • Merging the upload folders and files actions into one prompt
  • Folder-level permissions

As a result of our discussion, we agreed to move some of the enhancements to a V2. I updated the wireframes to limit the scope of actions and handed it off to Engineering.

The Final Design

the delivery and impact

Small Feature Enhancements, Big Increase in Productivity

The Data File enhancements were delivered in the Incorta 4.6 release (check out the release notes). Product Management shared positive feedback from users, saying that there was a sizable difference in the experience of their workflow. They were able to finally edit all files right inside the Incorta app, share top-level folders with peers, and cut down time by a lot. They also appreciated that the Data Files page was finally redesigned to be consistent with the rest of the platform.

Another achievement from this was that the breadcrumb design is going to be standard implementation. The next phase for this project is currently in the works, which includes folder-specific permissions and moving folders around.
