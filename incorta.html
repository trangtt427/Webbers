Incorta

Data File Directory

Product Design

The Incorta software allows for realtime data analytics without needing any transformation or data warehousing. Unfortunately, the data analysts using Incorta were dealing with lackluster data file support which made their job more grueling than it needed to be. I led the design of several features to help users better manage their data files and sources so that they can focus on their actual job: uncovering important business insights. The results were increased productivity for users and a high number of customers upgraded to the latest version of Incorta.

opportunity

The Villain of this Story: A Poor File Management System

Users have long complained about Incorta's lack of support for data file management. There was no concept of a folder directory, data files could only be uploaded one at a time (and couldn't be edited once brought in), and finding files was a nightmare (no native searching functionality). A workflow for users to tailor their Incorta environment was missing entirely.

The current Data page was one of the last pieces that needed to port over from Angular to React. The duality of this opportunity meant tackling an engineering initiative and a highly requested user ask in parallel.

the role of design

I led the design efforts to ensure that data analysts could get their time back, through quality-of-life improvements and an intuitive workflow.

This involved gathering user insights, partnering with Product for ideation and prioritization, running user studies in the Incorta Lab, refining the Incorta Design system, and working with Engineers to scope across multiple product releases.

research and discoveries

Understanding the Incorta Data Analyst

The PM and Solutions Architects frequently visited customers onsite and were the gateway to my learning about our users, the data analysts. Internally, these data analysts were considered our "power users". They use advanced analytical tools to dive into their data, working to create visualizations for their stakeholders. However, before that happens, data needs to be brought into their Incorta instance first.

User Story Map

These are the pathways of how data can be brought into one's Incorta environment.

There are two ways of bringing data into Incorta. First is by connecting to a database. Second is by uploading a data file. If going with the file upload route, users would have to go through a few hurdles if they're looking to manipulate their files.

The Frustrations of Data Files

Once a file has been uploaded, a user can't change its name.

The Workaround: change the file's name prior to uploading. The Catch: having to remember to do this every time the user uploads a file.

A user can only upload one file at a time.

No workaround. Pure deprivation.

No native search functionality.

The Workaround: cmd+F or ctrl+F. The Catch: a user would scroll down to load more files on the page. Using the search shortcut would only scan what's currently loaded on the page.

Files are laid out on one level. Folders are not supported.

The Workaround: name related files in a way where they can be near each other once uploaded. The Catch: extra upfront user effort.

Users have been asking for better navigation throughout the platform for a while.

Folders and file directory support were already implemented in the new React parts of the software, but users still had trouble traversing through folder levels. This was due to our poorly built navigation patterns. Users griped about our existing breadcrumb in the Content page (the only other page that contained files) because it didn't provide a way to browse through hierarchy of folders.

The current navigation pattern only showed the current file location.

Vision

We envisioned the Data Files structure to be redesigned as a directory with intuitive navigation.

Based on what we've been hearing from our users, we came up with the following criteria for success.

  1. Ability for users to create/modify folders
  2. Ability for users to upload folders
  3. Ability for users to manage permissions to overwrite
  4. Allow "New" button to combine actions for data file and data source
  5. Implement Folder permissions
  6. Ability for users to move files/folders
  7. Improved file navigation

design

Revisiting the Breadcrumb Navigation

I looked to a few big names in the storage service space like Google Drive, Onedrive, and Dropbox to see which use cases can be applied to our own. We recommended the common breadcrumb with a clear distinction of a user's location in a path, a pattern that users of modern day technology would recognize.

Scoping

The proposed designs satisfied user goals, but we were met with resourcing constraints.

In a sync with the Server and Front-end teams, we learned that they couldn't commit to some of our asks. Resources have been allocated to some of the higher priority backend projects. Some of the things they couldn't accommodate in the next two sprints included:

  • Functionality to move folders
  • Merging the upload folders and files actions into one prompt
  • Folder-level permissions

We agreed on moving these proposals out of our MVP to future releases so that we can prioritize getting this delivered on time.

the delivery and impact

The Final Design

Small Feature Enhancements, Big Increase in Productivity

After about two months, the Data File enhancements were shipped in the Incorta 4.6 release (check out the release notes). Our customers don't usually upgrade right away to the newest Incorta platform as it does take a bit of work. But they were willing to switch to Incorta 4.6 after hearing that Data File upgrades were a part of the package. After the release, we received feedback from users about how they can finally cut down some time around maintaining their data.
