Baby Design UI: A robust and easy-to-use Figma UI kit to help you start designing right away.

Summary

Built a UI kit that includes dozens of components and 600+ variants

Currently maintained with new components and variants continuously being added

En route to becoming a complete design system

Downloaded from the Figma Community 950+ times

Design systems bring clarity and velocity to designing products, but at one company I worked for, we wrangled with a design system that fell short of delivering that.

At a glance, the system contained all the essentials to make sure designs are consistent and polished. But when multiple designers started using it, we found that it wasn't built for scale. Things were constructed in a way where it wasn't easy to change between styles on the fly. We ended up slowly rebuilding parts the design system so that it would easier and faster to use in the long term.

The design system contained too many slightly different variations of components and not enough were easily configurable for our needs.

If you wanted to use small buttons of a certain state and color, you'd have to search for a separate component and take your best guess at the name. The design system was built in Figma, but unfortunately didn't use the variants feature to group components of the same properties.

Some of the existing components were created with extra layers just so they could be hidden or exposed in order to toggle between permutations.

After this experience, I was inspired to try my hand at creating a design system; one that is extensive, simple to use, and optimized for repeatability. This personal project became lovingly known as Baby Design UI.

I created components with as many customizable properties as possible to make it obvious that only a single component would be needed—one for buttons, one for input fields, etc.

Users would only need a single component if they wanted to control all styles for a button. This would take the guesswork out of knowing what was available.

The UI kit is modular; you can build upon existing components and fine tune nested things like tables.

Documentation

For Baby Design, I documented all the variants for each component. Colors, typography, icons, components and variants—everything is organized and outlined clearly so that people know exactly what they can do. Users would be able to navigate through quick links and customize base elements as they please.

The best design systems are intuitive, have a low barrier of entry, and strategically control their customizations. But it's also important to assess if one needs to be built.

It may not be cost-efficient for a new company to go all in on their own design system as one of their first pursuits. Using one of the many popular systems out there could be a viable option depending on the business needs. What's more important than knowing how to create a design system is knowing how to use one.
