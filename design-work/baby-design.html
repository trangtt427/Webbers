Baby Design UI

A rich and easy-to-use Figma UI kit to help designers build clean interfaces, quickly and consistently.

Summary

Created a UI kit that includes light and dark modes, dozens of components, and 600+ variants

Currently maintained with new components and variants continuously being added

Downloaded from the Figma Community 1k+ times

Design systems bring clarity and velocity to designing products, but I wrangled with one that fell short of delivering that.

I was using a design system that was spun up for a financial product that I was working on. At a glance, the system contained all the essential elements. But when multiple designers started using it, we found that it wasn't built for scale. Things were constructed in a way that wasn't straightforward for designers. We ended up slowly rebuilding parts the design system so that it would easier and faster to use in the long term.

The design system contained too many slightly different variations of components and not enough were easily configurable for our needs.

If you wanted to use small buttons of a certain state and color, you'd have to search for a separate component and take your best guess at the name. The design system was built in Figma, but unfortunately didn't use the variants feature to group components of the same properties.

Some of the existing components were created with extra layers just so they could be hidden or exposed in order to toggle between permutations.

After this experience, I wanted to try creating a design system; one that is extensive, simple to use, and optimized for repeatability. This personal project became lovingly known as Baby Design UI.

To remove the guesswork, I created components with as many customizable properties as possible.

Users would only need a single button component if they wanted to change its properties like color, size, and style. Same with input fields, checkboxes, radio buttons, etc. Designers don't have to look far for what they need.

The UI kit is modular; you can build upon existing components and fine tune nested things like tables.

Documentation

For Baby Design, I documented all the variants for each component. Colors, typography, icons, components and variants—everything is organized and outlined clearly so that people know exactly what they can do. Users would be able to navigate through quick links and customize base elements as they please.

Customizable Themes

Right after Figma announced their new variables feature at Config 2023, I felt it was the right to get with the program and introduce dark modes for Baby Design. Variables made it super easy to implement consistent colors and swap between them. Baby Design now comes in four themes: purple or blue, each as either light or dark mode.

Baby Design has gone to the dark side.

The best design systems are intuitive, have a low barrier of entry, and strategically control their customizations. But are they necessary?

It might not be the most cost-efficient for a company to go all in on their own design system if they don't have too many product offerings and need to build fast. Using one of the many popular systems out there is a great option depending on the business needs. Most of those systems have already been validated for almost every use case and are flexible enough to be personalized to your brand.
