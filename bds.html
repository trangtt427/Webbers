opportunity

A Baby is Born

A design system is a collection of reusable UI elements and patterns, accompanied by guidelines on how they should be used. Instead of reinventing the wheel each time a new feature comes along, product teams use design systems to create consistent experiences faster without compromising quality.

A company I worked for built a design system as part of a massive redesign, but as soon as multiple designers started working out of it, we learned that it wasn't as efficient as we thought when designing at scale. We ended up rebuilding some of the components as Figma variants to simplify the workflows. Throughout the process, I was becoming increasingly curious and inspired.

"What's it like to build a design system from nothing?"

I was compelled to explore this realm a bit more. I wanted to try my hand at creating my own design system from scratch; one that would help people design faster, is neatly organized, and has straightforward customizations. For a month, I dedicated a dozen hours a week towards this personal design system—which I lovingly dubbed Baby Design System.

solutions

One Component, Many Styles

My company's design system had multiple components for various states. For example, we had two different button components for size; one small and one big. Some components were made with extra layers so that we could hack different variations. I solved this in Baby Design System with variants and built in as many customizable properties as possible per component. My goal was to limit the components users had to think about and give them full control of the styles of those components.

Style just about anything for a button using a master component.

I applied the same principle to complex components such as tables. Properties like checkbox visibility and row height can be changed at the table level, but I've also exposed the ability to manage nested component properties. Row items can be swapped between text, actions, statuses, and avatars.

Tables can be customized down to each row item.

Documentation to Help You Find Your Way

Dabbling in the company's design system was a learning experience and reinforced the importance of detailed guidelines. Sometimes I'd stumble across a component that I should have used instead of the one I implemented. Other times, I'd be using the right component, but would have to fiddle with the layers to get the to the permutation I wanted. Having these things spelled would have saved me time, so when it was my turn to do this for Baby Design System, I documented all possible levels of customization offered. Colors, typography, icons, components and variants—everything was well-organized, in plain view, and had accompanying notes.

Organized and Easy to Make Changes

All of the styles are documented. Making changes to base elements is easy. Colors and text could be changed to proliferate throughout all of the components.

I used a plugin called Foundation: Color Generator to create a color palette and nicely plot the profiles in the documentation.

Variant Properties Shown for Every Component

Variant properties are listed for each component and users have shortcuts to jump to master components.

learnings

A Growing Baby

Nothing I've done here is groundbreaking, as this is typically how a design system comes to life in Figma. Is it a comprehensive system? Depends on what you need. There are lots of free design systems out there that are loaded with hundreds of components and styles. But if you want to design quickly and need just the basics—buttons, forms, alerts, tabs, even tables—then this design system is for you!

Creating (and even using) a design system properly requires discipline and planning. And as gratifying as this was, I think that any decision to create a design system at a company should be considered carefully. It may not be cost-efficient for a young company to go all in on their own design system as one of their first pursuits. Using one of the many popular established design systems tends to be the smarter choice for development velocity. What's more important than knowing how to create a design system is knowing how to use one and understanding the concept of building for modularity and repeatability.
