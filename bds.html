Baby Design System: A robust and easy-to-use Figma UI kit to help you start designing right away.

Summary

Built a design system that includes dozens of components and 600+ variants

Currently maintained with new components and variants continuously being added

Downloaded from the Figma Community 950+ times

Design systems bring velocity to designing products, but at one company I worked for, we wrangled with a design system that fell short of delivering that.

It was difficult to locate components and took some extra time to understand how some of them were meant to be used. Along the way, we sank time into rebuilding some parts of the design system so that it would easier and faster to use in the long term.

The design system contained too many slightly different variations of components and not enough were easily configurable for our needs.

At first, we weren't sure what styles and states were available to use since they all existed as separate components. For example, if we wanted to use small buttons of a certain state and color, there was no efficent way to know right off the bat if that was a customizable option.

Some of the existing components were created with extra layers just so they could be hidden or exposed in order to toggle between permutations.

After this experience, I was inspired to try my hand at creating a design system; one that is extensive, simple to use, and optimized for repeatability. This personal project became lovingly known as Baby Design System.

I created components to have as many customizable properties as possible to cut down the number of components users would have to deal with.

Users would only need a single component if they wanted to control all styles for a button. There would be no second guessing on what to use and how.

Documentation

For Baby Design System, I documented all possible levels of customization offered. Colors, typography, icons, components and variants—everything is organized and outlined clearly so that people know exactly what they can do. Users would be able to navigate through the file via anchor points and change up base elements as they please.

A design system is successful if there's a low barrier of entry to using it and that customization is strategically controlled but intuitive. It's also important to assess whether you need to build one in the first place.

It may not be cost-efficient for a new company to go all in on their own design system as one of their first pursuits. Using one of the many popular systems out there could be a viable option depending on the business needs. What's more important than knowing how to create a design system is knowing how to use one.
