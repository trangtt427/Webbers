Baby Design System

An Easy-to-use and Fully Customizable Design System

Contains the essentials for all of your rapid prototyping needs

Packed with customizable variants for each component, including hover and focused states

opportunity

A Baby is Born

A design system is a collection of reusable components and guidelines that help companies create products with velocity, consistency, and quality. Instead of creating components each time they build something new, designers use what they've already established. At one company, we had a design system in place that was used to develop a product a year ago, but once multiple designers were leveraging it for different products, we learned that the system wasn't battle-tested for managing design at scale.

I was spending a lot of time hunting for components in this system and trying to make sense of what I was supposed to use. I questioned why the components were constructed and organized in a certain way and thought about I would've done differently. This compelled me to take up the challenge. I wanted to create a personal design system from scratch, one that is loaded with the essentials, clearly documented, and easy enough for people to use right off the bat. For a month, I dedicated a dozen hours a week towards this new design system—which I lovingly named Baby Design System.

solutions

Accessibility is always on the brain

As I was starting with the basics of the design system—fonts, colors, icons—I wanted to make sure everything would have enough contrast and pose no accessibility issues. I've always had trouble coming up with color palettes (I used to not pay attention to hue, saturation, etc.), so I was happy that a Figma plugin called Foundation: Color Generator existed. I simply entered a color I wanted to use, and the plugin would generate different color profiles based on that color, including their color contrast ratios.

One Component, Multiple Variants

Instead of having multiple components for each of the permutations, I simply created a single master component, and then allowed flexibility to switch between the different types, sizes, states, icon visibility, etc. This makes it easy for designers to prototype. When searching within assets for a button, only one component will appear.

The one button component to rule them all!
