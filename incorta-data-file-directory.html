Incorta: Data File Directory

A design solution allowing for better file management in the Incorta app

Project Background

In the Incorta app, users can provide data in two ways: upload CSV/XML files straight into their instance or connect to a database. One of the biggest issues that users are facing is that they are adding hundreds of files into a root location, without capabilities of organizing any of them. The customer ask was that we provide a solution that allows basic directory capabilities instead of a flat level.

The old Data UI. It has not been migrated to React and did not provide a way for users to fully customize their environment.

Role

I led the UX design efforts for this feature. I worked with a PM, the engineering team, the server team, and collaborated with the lead UX designer occasionally for brainstorming and feedback.

Learning About the User

The persona of this feature is the business analyst. They are responsible for bringing the data into their systems in order to create schemas. Once all the data is there and usable, business users can then build visualizations.

I worked with the PM and a few solutions architects who speak directly to these power users to learn about their struggles and frustrations. Some of their pain points include:

  1. No ability to group files into folders
  2. Difficulty finding files because there's no way to search
  3. The "New" button is confusing because it has different actions depending on which tab you're on and there's no way for users to know that
  4. Want to upload whole directories

We realized that the user's goal is to save time and effort. If they can spend less time searching for files, then they can use that time to do other productive things within the app. Users have expressed their frustration when handling this part of the Incorta app. This solution will empower users to able to customize their environment in the Incorta app.

Planning

The PM wrote up the user story and specified that this feature would span across two releases. The first release would focus on these basic functions:

  1. Creating, uploading, and renaming folders
  2. Sharing files
  3. Searching
  4. Managing permissions to overwrite
  5. Allow "New" button to combine actions for data file and data source

The second release will address folder-specific permissions and how that experience will be for the users.

This feature would also require an enhancement in our navigation. Now that users will be able to nest folders, they'll need a way to traverse through previous folders. We currently don't show breadcrumb paths anywhere in our app, so this navigation would be an important component.

Prototyping the Solution

The proposal involves redesigning the existing Data tab to provide better support for folder structures and searching. At the time, I had already started up a design library for Incorta including the list and searchbar components. I used these components for the Data tab.

The only part of our app that utilizes a (half-baked) directory is the Content tab, where dashboards and folders can be grouped into further folders. However, the feature was poorly designed as there was no way for users to go back to the folders within a path.

Our current half-baked breadcrumb design.

Users couldn't see the previous folder, but also couldn't access the root location easily without clicking on the left carets each time. We've had complaints from customers and internal users about this scenario. I designed the breadcrumb to show the root location, an ellipsis revealing multiple folders, the folder right before, and the current folder that the user is in.

V1: The wireframe demonstrates the new breadcrumb navigation and directory design.

Users will be able to upload individual data files or whole folders.

The new design will support renaming of folders and various other functions.

After the implementation, we reviewed it in our test build. We didn't really like the idea that the title of the page was styled like the breadcrumb path. After some feedback and brainstorm sessions, we decided to separate the title of the page from the rest of the breadcrumb path, so that it is more obvious to the users.

V2: Second version with the improved breadcrumb.

Evaluating the Usability

This feature was a huge ask from multiple customers. After the feature was released, users were very pleased with the enhancement because it was a huge time and effort saver. They could now share folders containing appropriate files and search for files using a searchbar. Our customers were excited to hear that we were expanding on that design in the next release.

Process Overview

Define the problem

What actions are users trying to take with their data files?

Gather information

Researched current trends in designing directories, and worked with PMs to learn about user needs.

Ideation

Brainstormed all user scenarios to design experience.

Iteration

Changed breadcrumb design based on feedback.

Implementation

Took 2 months of design, development, and testing.

Evaluate

Customers shared feedback with us during PM visits.

The Outcome

The users were delighted with the first phase. The Data page was also one of the last pages that needed to be migrated to React, so it was a big accomplishment to get that done along with this directory enhancement.

The next steps for this project is currently in the works, which includes folder-specific permissions and bulk actions.
