Data File Directory

Product Design

Incorta 2019
Users of Incorta were adding large amounts of data files into a root location with limited file management capabilities. This long-awaited feature resolves that issue, granting Incorta users full control of their data workflows.

About Incorta

Incorta is a hyper-converged data analytics platform that removes the ETL process in data warehousing and lets users make use of their data fast. In the Incorta desktop app, users can provide data in two ways: upload CSV/XML files straight into their instance or connect to a database. From this information, relationships can be created in order to make dashboard visualizations and tabular reports.

The Villain of This Story: A Poor File Management System

The current Incorta Data Files page was one of the few remnants of the Angular framework which the platform was built on a few years ago. While it was time to port it over to React, there were other requested features in the roadmap calling for its enhancements. Currently, users couldn't sort and search within files, which are common capabilities you'd expect from a file management system. Users also wanted to be able to create folders to store related data files. I partnered closely with Product Management to work through the user pain points and discover the requirements needed in order to make this a robust data management system.

The old Data UI has not yet been migrated to React and did not provide a way for users to fully customize their environment

Understanding the Incorta Users' Workflow

While I worked mainly with the PM, I also had insights from stakeholders like solutions architects. They provided me with details about our users, which aligned with the pain points mentioned in the user stories. Our target persona, the Data Analysts, are the Business Users responsible for bringing data into their systems in order to create relationships between objects.

Persona: The Data Analyst

Trisha Johnson

40 years old

Occupation

Data Analyst

Story

As a power user of the Incorta app, Trisha manages the data. She connects databases, uploads files, and constantly shares files with other users. Trisha gets frustrated with not being able to categorize her data files. As a result, she is left to look for the files needed each time when she visits the Data tab.

Goals

Wants personalization. Wants to be able to customize her environment in a way where files are grouped into folders so that they're easier to to find and share amongst others in her company.

Behaviors and Beliefs

Very technical and loves being able to customize everything.

Some of the Data Analyst's pain points include:

  1. Can't group files for organization
  2. Difficulty finding files because there is no way to search
  3. The "New" button is confusing because it has different actions depending on which tab you're on and there's no way for users to know that
  4. Want to upload whole folders

We realized that the Data Analyst's goal is to save time and effort. If they can spend less time searching for files, then they can use that time to create joins between tables and rows or connecting to data sources. This solution will empower users to customize their environment within Incorta.

Brainstorming the Enhancements

The PM and I brainstormed ways to translate the requirements into features. On a whiteboard, we listed user actions and prioritized them based on how related they are to the urgent asks. Here are the tasks we came up with for quick wins:

  1. The ability to create folders and rename folders
  2. Uploading folders
  3. Searching and sorting
  4. Managing permissions to overwrite
  5. Allow "New" button to combine actions for data file and data source
  6. Folder permissions
  7. Move files/folders

This feature would also require an enhancement in our navigation. Now that users will be able to nest folders, they'll need a way to traverse through previous folders. We currently don't show breadcrumb paths anywhere in our app, so this navigation would be an important component.

Designing the New Breadcrumb

The only part of our app that utilizes a (half-baked) directory is the Content tab, where dashboards and folders can be grouped into folders. However, the current implementation doesn't provide a way for users to navigate back to folders in a path.

Incorta's current half-baked breadcrumb

Users couldn't see the previous folder, but also couldn't access the root location easily without clicking on the left carets each time. We've had ongoing complaints from customers and internal users about this experience.

Solution: I designed a breadcrumb component to show the root location, an option to look at multiple folders in that path, the previous folder, and the current folder that the user is in.

The breadcrumb navigation in the instance of many folders in a path

The Engineering Constraints

In a sync with the server and front-end teams, we found out that some of our tasks would not be able to fit in the upcoming release.

  1. The server team cannot support the relocating of folders because it was a huge engineering effort and they were slammed with higher priority tasks.
  2. The server team cannot support folder-specific permissions.
  3. The front-end team is unable to merge the upload folders and files into one prompt (their current technology doesn't support that).
What if a user doesn't have access to a certain folder, but has access to the file within it?
- Back-end Engineer

We realized the complexity of folders and permissions after meeting with the teams. As a result of our discussion, we agreed to remove some of the enhancements (moving folders and folder-specific permissions) to include them in the later release. For the first release, we are only attaching permissions to files instead of folders. I updated the wireframes to limit the scope of actions and handed it off to Engineering.

The Move icon was removed in the newest design since we didn't have enough engineering resources to work on that back-end support

Uploading data files and folders have been separated into two actions since engineering could not support a single dialog

Touching Up the Breadcrumb Again

After implementation, we reviewed the feature in our dev environment. In a design sync, we eventually agreed that the title of the page shouldn't be styled like the breadcrumb path. The feedback was that the page title should assume its own identity, thus it was decided to move the title of the page to the next line. It would also be helpful to users if the page title stayed in one spot and not having to jump around depending on how nested the file is.

The newest version of the breadcrumb that shows the title page in a separate style

The new structure for the Data tab along with the enhanced breadcrumb

Rolling Out the Feature

After it was QA'd and approved, the feature was rolled out to all users. We had positive feedback from users, despite not being able to truly treat the Data tab like a directory yet. We were told that as a result of the feature, it made a huge difference in their data management workflow. They also expressed their excitement for the upcoming enhancements in the second phase.

Phase 1 complete!

The enhanced flow of adding a data file or data source

The new Data UI built with React

We've been waiting for a long time to get searching and sorting on this page, and now it saves me so much time. I'm also glad that I can create folders to keep everything tidy.
- Business analyst

This support for data files was a long-awaited wish for the customers. Moving forward, the breadcrumb design is going to be the standard implementation for the other pages in Incorta whenever applicable. The next phase for this project is currently in the works, which includes folder-specific permissions and moving folders around.

The Takeaway: Thoughts and Lessons

Working on this feature was a reminder that different teams have different priorities. While this feature is an important one to the users, we were limited by constraints because the engineering teams were unable to support everything in the design. It was a strategic course of action to give the users what they asked for along with easy improvements, followed by bigger enhancements in the next release.
