Incorta

Design Library and Dark Mode

visual design | product design | design systems

With the promise of delivering a Dark Mode for the next release, it was important to first nail down our design system. This would solve the issue of not having a single source of truth for shared assets between teams. I was tasked with creating a living design library to unify all of our assets, while coming up with a fully accessible dark theme for our platform. The results amazed prospects and current users in feature demos, which contributed to successful closing of contracts.

opportunity

An Incorta Dark Mode was a roadmap item for the next release.

One of our longest tenured customers requested a dark version of the Incorta platform. We learned that they were interested in the feature for a very specific reason: the visualizations were a bit hard to see when presenting them in meetings with other key business members. A few of their users also expressed interest in a dark mode because their system preferences are set to a dark mode, so most of their apps followed suit. Having to switch over to the light UI of Incorta was a bit jarring to their eyes.

Product felt that it was an easy task to deliver but the issue at the time was that we hadn't yet codified a design system. Without one, it would be difficult to even begin thinking about a dark equivalent.

We used the Ant Design as building blocks for our components. We also had to alter some of the assets to align with Incorta-specific concepts and use cases.

"Hey, where can I find the right [insert component here]?"

In order to start working on an Incorta Dark Mode, we needed to first put together a design system. This was appropriately timed because we currently had problems that threatened the way designers and engineers collaborated: no alignment on where to store all of our design assets. Icons, fonts, and colors were scattered between Ant Design components, dozens of Sketch files and people's local desktops. There were a few issues with this:

  1. Engineers were not being informed of where to find certain assets, so they were implementing things outside of our Incorta colors and icons.
  2. The codebase became convoluted because we had too many variances of certain colors. It was easy to get colors and text styles mixed up.
  3. Designers were spending more time than necessary to create wireframes because we didn't have a master component library.

By finally cleaning up our design system and creating a symbol library, I'd be able to craft a dark theme for users while making processes efficient for both designers and engineers.

the role of design

I led the initiative of building up our design library and creating Incorta Dark Mode.

I unified all text styles, colors, created icons, and built modular components for our Sketch library based on the Ant React Library. I worked alongside a front-end engineer to provide design direction for Incorta Dark Mode and to make sure that their codebase reflected our current design styles.

research and discoveries

Auditing the Incorta Platform

I first went through the desktop app to tease apart the inconsistencies. I documented all the text styles, colors, border widths and radii, drop-shadows, and icons. One other thing to note was that Engineering was in the middle of migrating everything over from Angular to React in parallel of building new features.

Design Inconsistencies in the Incorta UI

We had about 15 different shades of grey.

There were multiple icons being used for the same action.

Too many font styles with subtle differences.

Inconsistent border widths and shadows.

"The amount of grays in our app is depressing." - Design Manager's response when finding out that we were using so many grays.

"Why are we using 3 different search icons?" - VP of Product

We accumulated so many variations of styles because some high fidelity mock-ups used incorrect hex codes and ended up going through development without being questioned We accumulated so many styles because we didn't have proper documentation and guidelines in place. Sometimes Design was not communicating to Engineering what icons should be used, so Engineering would have to resort to similar icons from the Ant library. These styles ended up going through development without being questioned.

Vision

The vision was to improve workflow between teams with a robust design system, and to help alleviate visual discomfort for users with a dark interface.

Criteria for Success

  1. Design Library complete with a Master Sketch file
  2. Flexible Dark Mode components

design

Typography

The approach was to first focus on the basic building blocks, starting with text. After collecting all of our styles, I shared the findings with the Design team to determine which we wanted to keep. I wrote guidelines on how to implement the text, such as spacing, height, alignment, and casing.

Colors

We investigated where certain colors were being used and which ones should be omitted. After getting a consensus on which ones we should keep, I wrote up some guidelines on how to use them. Between similar colors, we tested which colors had better web contrast ratios.

The plan was to make sure whatever colors we picked for the main content of Dark Mode would have to accommodate this set of colors.

Icons

In our design sync, we narrowed down some of the icons we wanted to proceed with. In some instances, we didn't have Ant icons to represent some Incorta concepts. I had to find existing icons and redesign them in Illustrator to accommodate our use. I also altered several icons to make them appear as if they came from a family of icons.

The plan was to make sure whatever colors we picked for the main content of Dark Mode would have to accommodate this set of colors.

Building the Components

The last phase of the design library was to create the components. I collected the UI patterns in our app and created Sketch symbols using our updated colors, fonts, and icons. They were categorized by their usage, whether they were navigation-specific symbols, table components, or used globally.

The master Sketch file was shared amongst the other designers to use for building prototypes quickly.

Going to the Dark Side

With the design system completed and approved, it was time to start building out the dark UI. I mapped out every possible Incorta scenario so I can explore various colors to ensure that everything sat well. Things I noticed during my exploration:

  • Elevation doesn't work quite as well with a black background
  • Overlays can't be too dim
  • You can't simply just invert colors

the delivery and impact

Introducing the Incorta Design Library and Incorta Dark Mode

Internally, we referred to our design system as "Dumplings".

The design system helped control consistency between teams. Engineers now had a codebase that was updated with our new design styles. Having the design system readily available as a Sketch library saved time as we were able to pull the assets directly into our prototypes. We were no longer opening separate Sketch files to find out how something was designed, where to find an icon, or what color to use.

Dark Mode was rolled out to all customers as an Incorta Lab Feature in Incorta 4.6 (check out release notes). Despite being created to support one customer request, plenty of users began using Incorta in Dark Mode. Users applauded the theme as refreshing. The Incorta desktop app now sat nicely with the rest of other dark mode apps.

"🔥🔥🔥" - A customer's reaction during our feature demo
